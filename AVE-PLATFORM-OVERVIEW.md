## Architecture Principles

- **Stateless by default** — In-memory stores with production-ready persistence notes throughout
- **Every service has** `/health` — Docker Compose healthchecks on every container
- **Every service has** `/v1/analytics` — Built-in observability and operational metrics
- **Every service has** a `README.md` — Full API reference with endpoint specs and data models
- **CORS enabled** — All services accept cross-origin requests for dashboard integration
- **Pydantic validated** — Strong input validation on every endpoint
- **One compose file per phase** — `docker-compose.phase{N}.yml` for independent deployment
